Gone are the days when you had to remember several schemas and codes to efficiently manage data operations. You can now make your data consumption-ready, for all your analytical activities, using the StreamSets Data Collector (SDC): a data operations platform.

With the SDC, you can move batch or streaming data from one location to another using dataflow pipelines. You need minimal or no coding knowledge to build, test, and operate dataflow pipelines. This is because the SDC offers you a graphical user interface (GUI) to drag and drop processors that work on your data.

The SDC could be considered as an alternative to Apache NiFi, which is also available in NSC. Please refer to the blog on Apache NiFi to learn how to use NiFi in NSC.

What makes the SDC worth trying?

Its the ability to process binary data and support the whole file data format.

Before we get into how to use the SDC in NSC, lets quickly recap a few key concepts. Skip this section if youre well up to speed and just want to know about the NSC implementation.

 

Before we use the SDC in NSC, lets check if we have the basic things ready.

       Ensure that you have the following minimum configuration available in your NSC quota, to run the SDC: - CPU: 4, RAM: 4GB

 

       Ensure that you have a persistent MapR volume in NSC. If you dont have a persistent volume, create a MapR persistent volume with a specified name, and of the required size in NSC. Please refer to the NSC User Guide to learn how to add a volume in NSC.

 

Your persistent data is stored in the following folder: /persistent/nsc_sdc/<YourUsername>/<YourDeploymentNamespace>/

 

The persistent data stored here include the pipeline data, the SDC configurations, the SDC resources files, and the SDC staging libraries. You can access these data directly through the SFTP client. The rest of the setup is the same as the StreamSets official setup that you will be familiar with. Please refer to the StreamSets user guide to learn more.

 

Lets now consider what needs to be done to use the SDC in NSC. Follow these steps:

 

1.       Lets call our persistent MapR volume samplevol.

 

2.       From the NSC Generic Service Catalogue, deploy an SFTP instance to obtain access to the files in the newly created volume. Please refer to the NSC User Guide to learn how to deploy a service in NSC.

 

3.       In the Deploy Service window, fill in the required details and click Submit. Make sure you select the persistent volume for deployment.

 

Note: You may choose any password as your Service Password.

 

 

Figure 2: Parameters to deploy SFTP for SDC

4.       From the Deployments tab, click the SFTP link displayed in your deployment record, and copy the host and port number given in the link to your SFTP client to access the volume. You may ingest data into this volume to execute your SDC jobs. Please refer to the NSC User Guide to learn how to access a volume and how to ingest data into an NSC volume.

5.       Deploy the SDC service from the Generic service catalog.

 

 

Figure 3: Deploying SDC

6.       In the Deploy Service window, specify the required details and click Submit.

a)      Specify 4 CPU and 4 GB RAM as the Namespace Quota.

b)      Specify samplevol as the MapR volume.

c)      Specify the other details just as you do for other services.

 

 

Figure 4: Parameters to deploy the SDC

7.       To access the SDC UI, click the SDC link (shown below) on the Deployments tab.

 

Note: You will be able to view the SDC link only after the deployment status changes to ACTIVE. Please give the NSC system three to five minutes to deploy the service for you.

 

Figure 5: Active SDC deployment

8.       To log in to the SDC, use admin as your username as well as your password.

 

 

Figure 6: Logging in to SDC
